## Why this sub-spike exists

During step b of S10's physical gate walkthrough, the user flagged that the LA (lightning arrester) toggle should not be gated on Basic — LA is part of `plant_layout`, a Basic-tier feature per the `renewable_energy` seed. Investigation revealed a structural issue rather than a local bug:

- S7 invented feature-key names (`cables`, `energy`, `obstructions`, `icr_drag`, `dxf`) that don't exist in the backend seed. Only `plant_layout` aligned.
- S10 extended this onto two new surfaces (VisibilitySection + SummaryPanel PRO_PLUS rows) using the same invented keys.
- In production, every user — including Pro Plus — would see "upgrade" chips on features they've paid for, because `availableFeatures` returned by the backend never contains `cables`.

ADR-0005 captures the full root cause analysis. S10.2 is the corrective work.

## Scope

### In scope

1. **Typed registry.** New module `packages/entitlements-client/src/feature-keys.ts` with `FEATURE_KEYS` const map and `FeatureKey` union type (the six seed keys: `PLANT_LAYOUT`, `OBSTRUCTION_EXCLUSION`, `CABLE_ROUTING`, `CABLE_MEASUREMENTS`, `ENERGY_YIELD`, `GENERATION_ESTIMATES`). Re-exported from the package root.
2. **Narrow APIs.** `FeatureGate`, `useHasFeature` accept `FeatureKey` instead of `string`.
3. **Replace invented keys at call sites (S10-adjacent only):**
   - `VisibilitySection` "Show lightning arresters" — **ungate** (LA is part of `plant_layout`, Basic-tier).
   - `VisibilitySection` "Show AC cables" — gate on `FEATURE_KEYS.CABLE_ROUTING`.
   - `SummaryPanel` DC cable length / AC cable length rows — gate on `FEATURE_KEYS.CABLE_MEASUREMENTS`.
   - `SummaryPanel` AC capacity / DC-AC ratio rows — gate on `FEATURE_KEYS.ENERGY_YIELD`.
   - **`LayoutPanel` "Calculate cables" toggle — gate on `FEATURE_KEYS.CABLE_ROUTING`.** Added after S10-gate step (e) surfaced that a Basic user could enable cable compute. Disabled switch + "Pro" chip on the row label for Basic; plus a belt-and-braces submit-time coercion that forces `enable_cable_calc = false` when the user isn't entitled (guards against stale persisted params after a license downgrade).
4. **PREVIEW_ENTITLEMENTS overhaul.** Replace the single "all invented keys" preview with three plan-accurate variants: `PREVIEW_ENTITLEMENTS_BASIC`, `PREVIEW_ENTITLEMENTS_PRO`, `PREVIEW_ENTITLEMENTS_PRO_PLUS`. Preview license key routes to the right one so tier-switching no longer requires re-keying the app with a real license.
5. **Contract test.** New test in `entitlements-client` that asserts `ALL_FEATURE_KEYS` ⊆ the seed's key set (`seed-products.ts`). Runs under `bun run test`.
6. **Remove stale invented keys.** `icr_drag`, `dxf`, `obstructions` stripped from `PREVIEW_ENTITLEMENTS`. None of these map to features that will be gated in the future per the locked product decisions (see below).
7. **Process updates:**
   - `docs/principles/external-contracts.md` — new file holding the full "External contracts bind before code" principle, post-mortem of the S7/S10 incident, and the authoritative source-of-truth file table.
   - `CLAUDE.md` §2: short summary pointer to the principles doc + ADR-0005.
   - `CLAUDE.md` §7: one-line pointer to the principles doc (keeps §7 narrative lean).
   - `CLAUDE.md` §13: session-start step for entitlement-touching spikes.
   - `docs/SPIKE_PLAN.md`: S10.2 entry inserted; S11 / S12 / S13 entries amended with the locked gating decisions; cross-cutting criterion links to the principles doc.
   - `docs/adr/README.md`: index ADR-0005; re-assign S10.5's drawing ADR to ADR-0006.
8. **Product decisions locked in S10.2 (see [SPIKE_PLAN.md cross-cutting §](../SPIKE_PLAN.md) and [ADR-0005 §9](../adr/0005-feature-key-registry.md)):**
   - All export formats (DXF, KMZ, PDF, CSV) — **ungated** in S12 and S13.
   - ICR drag in S11 — **ungated**.
   - Obstruction drawing in S11 — gated on existing `FEATURE_KEYS.OBSTRUCTION_EXCLUSION` (Basic-tier).
   - Zoom / pan / undo — always ungated.
   - Revenue lever is (a) feature keys for *what gets computed* and (b) the `calculations` quota for *how many times*.
8. **Test updates.** `VisibilitySection.test.tsx` and any other S10 test referencing the old `"cables"` / `"energy"` literals point at the new registry constants. Fixtures updated.
9. **s10.md gate memo amendment.** Add a short note at the top referencing S10.2 + ADR-0005 so the S10 gate memo reads correctly in the final state.

### Out of scope

- **New product features.** No new UI, no new gates that S10 didn't already scope.
