---

## Overview

Merge `apps/mvp_dashboard` into `apps/mvp_web` so that one Next.js app at `solarlayout.in` serves both the marketing site and the Clerk-authenticated dashboard. This follows the proven pattern in `apps/web` (cloud product) which uses `(marketing)` and `(main)` route groups in a single app.

**Why:** Spike 4 introduced `apps/mvp_dashboard` as a separate Next.js app at `dashboard.solarlayout.in`. In hindsight, this creates unnecessary operational overhead (separate Vercel project, separate domain, separate Clerk app config) without architectural benefit. The cloud product `apps/web` demonstrates that marketing and authenticated pages coexist cleanly in one app.

## Changes in Detail

### 1. Root layout (`app/layout.tsx`)

**Current:** Renders `<Header>`, `<main>`, `<Footer>`, `<MvpToaster>` directly. No providers. No Clerk.

**After:** Provider stack wrapping just `{children}` and `<Toaster>`. No Header/Footer — those move into the `(marketing)` layout.

```
ClerkProvider (afterSignOutUrl="/")
  → html (suppressHydrationWarning for next-themes)
    → body
      → ThemeProvider
        → QueryProvider
          → TooltipProvider
            → {children}
            → Toaster (from @renewable-energy/ui/components/sonner)
```

### 2. Marketing layout (`app/(marketing)/layout.tsx`)

New file. Wraps the existing `<Header>` + `<Footer>` around marketing pages. All existing pages move into this route group — URLs remain unchanged since Next.js route groups don't affect URL paths.

### 3. Move existing pages into `(marketing)/`

All current pages at `app/page.tsx`, `app/about/`, `app/contact/`, etc. move into `app/(marketing)/`. Their tests move with them. No content changes — just a directory restructure.

### 4. Dashboard layout (`app/(main)/layout.tsx`)

Moved from `apps/mvp_dashboard/app/(main)/layout.tsx`. Uses `SidebarProvider + DashboardSidebar + SidebarInset` with the header bar containing `SidebarTrigger`.

### 5. Dashboard pages

Moved from `apps/mvp_dashboard/app/(main)/`:
- `page.tsx` → `app/(main)/dashboard/page.tsx` (download cards)
- `plan/page.tsx` → `app/(main)/dashboard/plan/page.tsx`
- `usage/page.tsx` → `app/(main)/dashboard/usage/page.tsx`
- `license/page.tsx` → `app/(main)/dashboard/license/page.tsx`

All corresponding tests move with their pages.

**URL adjustment:** The `DashboardSidebar` nav item hrefs change from `/`, `/plan`, `/usage`, `/license` to `/dashboard`, `/dashboard/plan`, `/dashboard/usage`, `/dashboard/license`.

### 6. Sign-in / Sign-up pages

Moved from `apps/mvp_dashboard/app/sign-in/` and `app/sign-up/`. The `<SignIn>` component's redirect stays at `/dashboard`. The `<SignUp>` component redirects to `/dashboard`.

### 7. Clerk middleware (`middleware.ts`)

New file at `apps/mvp_web/middleware.ts`. Protects `/dashboard(.*)` routes. Unauthenticated users hitting `/dashboard` are redirected to `/sign-in`. Signed-in users visiting `/sign-in` or `/sign-up` are redirected to `/dashboard`.

### 8. Header update

Add Clerk auth buttons using `<SignedIn>` / `<SignedOut>`:
- Unauthenticated: Show "Sign In" button (links to `/sign-in`)
- Authenticated: Show "Dashboard" button (links to `/dashboard`)
- Desktop CTA "Download Now" remains as-is
- Mobile sheet nav: add Sign In / Dashboard link at the bottom
