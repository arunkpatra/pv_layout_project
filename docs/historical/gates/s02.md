# S2 — FastAPI sidecar: health, schemas, auth · Gate Memo

**Spike:** S2
**Date:** 2026-04-23
**Status:** awaiting human gate verification

## What was built

A working HTTP sidecar bound to loopback, token-gated, with every pydantic schema that mirrors `pvlayout_core.models.project` registered and visible in Swagger. `/health` and `/openapi.json` work end-to-end; echo endpoints validate each schema round-trip.

### 1. Runtime configuration — `pvlayout_engine/config.py`

- `SidecarConfig` dataclass carrying `host`, `port`, `token`, `version`.
- Environment-variable overrides for each, with sensible dev fallbacks:
  | Env var | Fallback |
  |---|---|
  | `PVLAYOUT_SIDECAR_HOST`  | `127.0.0.1` (always — we never bind non-loopback in v1) |
  | `PVLAYOUT_SIDECAR_PORT`  | OS-chosen free port via `bind(0)` |
  | `PVLAYOUT_SIDECAR_TOKEN` | `secrets.token_urlsafe(32)` (fresh per launch) |
  | `PVLAYOUT_VERSION`       | `__version__ + "+git." + short_sha` if in a git checkout, else `__version__` |
- Token shorter than 16 chars is rejected at startup. Constant-time comparison via `hmac.compare_digest`.

### 2. Pydantic schemas — `pvlayout_engine/schemas.py`

Pydantic v2 mirrors for every dataclass in `pvlayout_core.models.project`, using `extra="forbid"` so client typos surface as 422s:

- **Enums:** `DesignType`, `Orientation`, `DesignMode` (string-valued).
- **Inputs:** `ModuleSpec`, `TableConfig`, `LayoutParameters`.
- **Placed objects:** `PlacedTable`, `PlacedRoad`, `PlacedICR`, `PlacedStringInverter`, `CableRun`, `PlacedLA`.
- **Energy model:** `EnergyParameters`, `EnergyResult`.
- **Bundle:** `LayoutResult` (with `usable_polygon` intentionally omitted — it's a shapely runtime artifact reconstructed server-side each call; never travels over HTTP).
- **Sidecar-specific:** `HealthResponse`, `ErrorResponse`.

A `SCHEMAS_FOR_INSPECTION` registry maps url-slug → class for the dev echo routes.

### 3. FastAPI app — `pvlayout_engine/server.py`

- `build_app(config: SidecarConfig) -> FastAPI` factory.
- Bearer-token dependency on every non-public route (`/health`, `/_schemas/echo/*`); applied via a sub-router so `/docs`, `/redoc`, `/openapi.json` stay public.
- 401 with `WWW-Authenticate: Bearer` header for missing/malformed/wrong tokens.
- CORS middleware permissive (token auth is the real gate; CORS is a dev-tool affordance).
- HTTP request logging middleware (minimal — method/path/status).
- `GET /health` → `{"status": "ok", "version": "..."}`.
- `POST /_schemas/echo/<slug>` for each entry in `SCHEMAS_FOR_INSPECTION` — validates payload against the corresponding schema, echoes it back. Surfaces all schemas in `/docs`.

### 4. Entry point — `pvlayout_engine/main.py`

```bash
uv run python -m pvlayout_engine.main
```

- Resolves config, builds app, emits one line to stdout:
  `READY {"ready": true, "host": "127.0.0.1", "port": 56692, "token": "...", "version": "0.0.0+git.8b352b7"}`
- `READY` prefix lets the parent (Tauri shell, dev scripts) `startswith` filter unrelated log lines.
- Then hands control to `uvicorn.run(host=..., port=..., access_log=False)`.

### 5. Tests — `tests/`

- **`tests/smoke/test_schemas.py`** (8 tests, 6 skipped for no-default schemas): every registered schema builds + JSON round-trips; positional schemas round-trip; `LayoutParameters` defaults match domain; extra fields get 422; `LayoutResult` never serializes `usable_polygon`; nested schemas resolve.
- **`tests/integration/test_server.py`** (18 parameterised tests): `/health` 401/200, `WWW-Authenticate` header present, `/docs` + `/openapi.json` stay public, OpenAPI registers every schema + route, each echo endpoint round-trips a valid payload, extra fields → 422, unauth echo → 401.

All 34 tests pass (6 skipped for required-field schemas, covered by the positional round-trip test).

## Known limitations (by design)

- **No business logic yet.** Every route either echoes or returns a static health dict. `/parse-kmz`, `/layout`, `/refresh-inverters`, `/export/*` all land in S3+.
- **Echo endpoints are temporary.** They disappear when S3 replaces them with real routes. Their only purpose is to surface every schema in Swagger so gate #4 can be verified visually.
- **Token is printed on stdout.** By design — the Tauri shell captures stdout to discover the connection details. Any future change that has the sidecar log its stdout to disk must explicitly redact the token.
- **No `/graceful-shutdown` endpoint.** Tauri will SIGTERM the child on window close; uvicorn handles SIGTERM cleanly out of the box.
- **ezdxf still emits `PyparsingDeprecationWarning`** on import. Not ours; harmless.

## Decisions made during this spike

1. **Free port selection via `socket.bind(0)` then close, then uvicorn binds.** There is a tiny TOCTOU window on localhost that in practice has never bitten me; if we observe flakes in CI we'll add a retry loop around the uvicorn bind.
2. **Token generation default is per-launch `secrets.token_urlsafe(32)`.** Production usage always has Tauri set `PVLAYOUT_SIDECAR_TOKEN`; the random default is only for `uv run` dev.
3. **`LayoutResult.usable_polygon` omitted from the wire schema.** Shapely objects are not JSON-serializable and we never send them across the process boundary. Each sidecar call that needs the polygon reconstructs it from `boundary_wgs84` + roads.
4. **`extra="forbid"` on every schema.** Client typos fail loudly at 422 instead of silently dropping fields.
5. **Echo routes use explicit `__annotations__` assignment.** FastAPI's signature introspection via `typing.get_type_hints()` reads `__annotations__`; a closure-captured `cls` annotation does not resolve to the concrete class. This pattern is a one-time hack for the dev endpoints; S3's real routes have concrete annotations.

No ADR triggered.
