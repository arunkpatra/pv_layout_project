## What was built

### 1. ADR 0002 — canvas-first MapLibre, no basemap tiles

Original S8 plan expected evaluating MapTiler / Stadia / Protomaps / offline pmtiles for a basemap. Revisiting the design references (Claude Desktop, Linear, Figma — all basemap-less canvases) and PVlayout_Advance's matplotlib-on-white baseline flipped the default: **no basemap**. MapLibre renders our KMZ overlay on a solid `--surface-canvas` background. Doc: [`0002-no-basemap.md`](../adr/0002-no-basemap.md). SPIKE_PLAN / ARCHITECTURE / CLAUDE references all updated.

### 2. MapLibre vector styles

- **[`apps/desktop/public/map-styles/pv-light.json`](../../apps/desktop/public/map-styles/pv-light.json)** — minimal style: canvas background + boundary fill/outline + obstacle fill/outline + line-obstruction dashed stroke. Colours mirror `:root` canvas-vis tokens in [`tokens.css`](../../packages/ui/src/tokens.css).
- **[`apps/desktop/public/map-styles/pv-dark.json`](../../apps/desktop/public/map-styles/pv-dark.json)** — draft of the same six layers with `[data-theme="dark"]` token values. Polish deferred to S13.5.
- Three pre-declared GeoJSON sources (`kmz-boundaries`, `kmz-obstacles`, `kmz-line-obstructions`) with empty FeatureCollections so the React side can push data via `setData()` without inventing sources at runtime.
- No glyphs, no sprite, no tile endpoints.

### 3. `MapCanvas` composition rewritten

[`packages/ui/src/compositions/MapCanvas.tsx`](../../packages/ui/src/compositions/MapCanvas.tsx) replaces the S6 CSS-dot-grid stub with a real `maplibre-gl` instance:

- Init once on mount; `useTheme()` drives `map.setStyle()` on subsequent theme flips (initial mount is skipped to avoid a duplicate `setStyle` race).
- Props `boundariesGeoJson` / `obstaclesGeoJson` / `lineObstructionsGeoJson` push into the pre-declared sources via `hydrateSources`; empty FCs clear them.
- `fitBounds` centres the KMZ on load. A ref-keyed guard prevents re-fitting on every prop-identity change; viewport is preserved across theme swaps.
- `maplibregl.ScaleControl` bottom-left, metric.
- `attributionControl: false` (no tiles → no attribution). `dragRotate: false`, `pitchWithRotate: false` — a PV site is flat; rotation is noise.
- `children` still renders as absolute overlays (CommandBarHint, EmptyStateCard, error/loading overlays).

### 4. `@solarlayout/sidecar-client` extended

[`packages/sidecar-client/src/index.ts`](../../packages/sidecar-client/src/index.ts):
- `parseKmz(file: Blob | File, filename?: string): Promise<ParsedKMZ>` — builds the multipart FormData expected by the sidecar's `/parse-kmz` route (S3).
- New exported types mirroring the pydantic schemas: `Wgs84Point`, `ParsedBoundary`, `ParsedKMZ`.
- `extractError(body)` helper pulls FastAPI's `detail` string (and our own `{error}` shape) so `SidecarError.message` is the human-readable server message, not a generic `"Sidecar /X returned 422"`.

### 5. KMZ load flow

- **[`apps/desktop/src/project/kmzLoader.ts`](../../apps/desktop/src/project/kmzLoader.ts)** — `openAndParseKmz(sidecar)`: shows the native file dialog (`@tauri-apps/plugin-dialog`), reads bytes (`@tauri-apps/plugin-fs`), builds a `Blob`, POSTs to `/parse-kmz`. Returns `{ parsed, path, fileName }` or `null` on user cancel. No-op outside Tauri.
- **[`apps/desktop/src/project/kmzToGeoJson.ts`](../../apps/desktop/src/project/kmzToGeoJson.ts)** — pure `ParsedKMZ → { boundaries, obstacles, lineObstructions }` FeatureCollections. Closes open rings defensively. Also exports `countKmzFeatures()` for the status-bar text.
- **Rust side** — `tauri-plugin-dialog` + `tauri-plugin-fs` added in [`Cargo.toml`](../../apps/desktop/src-tauri/Cargo.toml), initialised in [`lib.rs`](../../apps/desktop/src-tauri/src/lib.rs). Capability: `dialog:default` and `fs:allow-read-file` scoped to `$HOME`, `$DOWNLOAD`, `$DESKTOP`, `$DOCUMENT`, `$RESOURCE` paths.

### 6. App shell integration

[`apps/desktop/src/App.tsx`](../../apps/desktop/src/App.tsx):
- New `project` state (`{ kmz: ParsedKMZ, fileName: string } | null`). Memoised `projectGeoJson` + `projectCounts`.
- `sidecarClient` promoted to a `useMemo` — a single typed instance shared across the app (parse-kmz now, layout / usage-report later).
- `handleOpenKmz` is wired in **four** entry points, matching the SPIKE_PLAN in-scope list:
  1. `⌘O` global keyboard shortcut.
  2. Native File menu → `menu:file.open_kmz` Tauri event listener.
  3. Command palette `Open KMZ…` entry.
  4. EmptyStateCard's `Open KMZ` button (a new `onOpen` prop on the composition).
- Top-bar breadcrumb now reads `project.fileName` once loaded.
- Status-bar `leftMeta` reads `"{N} boundary/boundaries · {M} obstacle/obstacles"` after load; pluralisation via a small helper.
- Loading overlay ("Parsing KMZ…") and error overlay ("Couldn't open KMZ" with Dismiss + Try again) render inside the MapCanvas during `opening` / `openError` states.
- InspectorSkeleton copy updates post-load ("Configure parameters and click Generate to place tables, ICRs, and cables.").

### 7. Bug fixes during implementation

- **MapLibre style validation** — initial drafts had `line-join` and `line-cap` under `paint` instead of `layout`, producing six errors per layer per style load. Moved to `layout`. Console now clean.
- **Duplicate `setStyle` on mount** — the theme-swap `useEffect([styleUrl])` fired on the initial render *after* the init effect had already loaded the same style, triggering a redundant rebuild plus its own warning. Gated with an `initialStyleUrl` ref so theme swap only runs on *subsequent* changes.
