## What was built

A monorepo scaffold with:

- **Bun workspaces** rooted at `pv_layout_project/` (`apps/*`, `packages/*`).
- **Turborepo** pipeline covering `build`, `lint`, `typecheck`, `test`, `dev`, `clean`.
- **TypeScript** base config at root, extended by each workspace package as they're built out in later spikes.
- **Prettier** (no semicolons, double quotes, es5 trailing commas, 80 cols) + `.prettierignore`.
- **EditorConfig** for cross-editor consistency (2-space JS/TS, 4-space Python/Rust).
- **`.gitignore`** covering node/Turbo/Next/Tauri/Rust/Python/PyInstaller/editor artifacts.
- **Workspace stubs** (package.json + README only; scripts echo and exit 0):
  - `apps/desktop/` — Tauri + React app shell (filled in S5/S6).
  - `packages/ui/` — shadcn-based component library (filled in S6).
  - `packages/sidecar-client/` — generated TS client from FastAPI OpenAPI (filled in S5).
  - `packages/entitlements-client/` — hand-written client for api.solarlayout.in (filled in S7).
- **Python sidecar skeleton** at `python/pvlayout_engine/`:
  - `pyproject.toml` with hatchling build backend, `requires-python = ">=3.12"`, ruff + pytest config, no runtime deps yet.
  - Empty `pvlayout_engine/__init__.py` with docstring and `__version__ = "0.0.0"`.
- **Docs infrastructure**: `docs/gates/STATUS.md` (this project's live status), `docs/adr/README.md` (ADR template + expected ADR index).

## How to verify (Human Gate)

Run the following from `/Users/arunkpatra/codebase/pv_layout_project`:

```bash
# 1. JS/TS install — should complete with no errors
bun install

# 2. Pipeline gates — all should exit 0 with stub output
bun run lint
bun run typecheck
bun run build

# 3. Python sidecar — should create .venv with no deps
cd python/pvlayout_engine
uv sync
cd ../..

# 4. Tag check — should show v0.0.0-s0 on HEAD
git log --oneline --decorate -n 5
```

**Pass criteria:**
1. `bun install` exits 0; `node_modules/` populated; `bun.lockb` present.
2. `bun run build` (and lint, typecheck) exits 0 for every workspace. You'll see `[S0 stub] <pkg> build` lines.
3. `uv sync` exits 0; `.venv/` appears under `python/pvlayout_engine/`.
4. `git log --decorate` shows `v0.0.0-s0` tag on the latest commit.

## Known limitations (by design)

- Scripts in every TS workspace are **echo-stubs** — they don't actually lint/typecheck/build anything because there's nothing there yet. Real linters and type-checks activate as each workspace gets code (S2+, S6+).
- No ESLint flat config at root — that lands in S6 when we have real TypeScript code to lint.
- No tests in the JS/TS side — Vitest lands in S6+.
- Python sidecar has **zero runtime dependencies**; `uv sync` creates an empty venv. Dependencies arrive in S1 (domain libs) and S2 (FastAPI).
