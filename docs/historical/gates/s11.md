## 1. Summary

S11 ships the two interactive features that make the app feel like a real layout tool: **drag an ICR to reposition it, or draw a rectangular obstruction.** Both commits dispatch a mutation to the sidecar, the preview stays visible on the canvas during the round-trip, and the layout atomically swaps to the recomputed state when the response lands.

Built against the design locked in S10.5 (ADR-0006 + design spec). Implementation followed the S10.5 demo's pattern with one extension: the `awaiting-ack` mode state, which preserves the visible preview between mouseup and sidecar ack so the UX doesn't snap-back-then-swap.

## 2. What was built

### Sidecar (Phase 1 — commit `cd81da5`)

Three endpoints, stateless (client owns `LayoutResult` and round-trips it), WGS84-in-UTM-out via `utm_epsg` on the wire:

- `POST /add-road { result, params, road: { road_type, coords_wgs84 } }` — projects WGS84 → UTM, appends to `placed_roads`, recomputes tables + LAs + inverters, returns updated `LayoutResult`.
- `POST /remove-road { result, params }` — pops `placed_roads[-1]` (LIFO, matches legacy "Undo Last"), 422s if empty.
- `POST /refresh-inverters { result, params, icr_override? }` — existing endpoint extended with `icr_override: { icr_index, new_center_wgs84 }`. Projects the WGS84 centre to UTM, shifts the ICR's bottom-left so the centroid lands at the requested point, recomputes.

**Critical correctness fix bundled in:** `/refresh-inverters` now ALWAYS calls `recompute_tables` before `place_lightning_arresters`. `la_manager`'s step-2 coverage check iterates `placed_tables`; running it on an already-LA-reduced set produces a subtly different LA placement than a fresh `/layout` would. Legacy parity requires `tables_pre_icr` as the source of truth. Without this fix, `/layout` and `/refresh-inverters` diverged silently for the same input — the integration tests caught it immediately.

**Tests (Phase 1):** 8 integration tests in `tests/integration/test_s11_endpoints.py` — full `/parse-kmz → /layout → S11-endpoint` chain against real `phaseboundary2.kmz`. Covers happy paths, schema rejections (missing `utm_epsg`, sub-3-vertex coords, icr_index out of range), empty-stack 422, backward-compat for `/refresh-inverters` without override, and the LA-drift-tolerant `add → remove` round-trip.

### Frontend (Phase 2 — commit `5e16e39`)

Full interaction stack, built on the S10.5 demo patterns:

- **`apps/desktop/src/canvas/`** — `debug.ts` probe factory, `coords.ts` Haversine helpers, `preview.ts` direct-to-MapLibre writers, `InteractionController.ts` mode router, `modes/icrDrag.ts`, `modes/rectDraw.ts`.
- **`apps/desktop/src/state/editingState.ts`** — Zustand slice with `EditingMode` union including new `awaiting-ack` value. Per S10.5 lesson: slice holds mode + session boundaries + undoStack; transient per-pixel preview bypasses Zustand to avoid React-render jitter.
- **`useRefreshInvertersMutation`, `useAddRoadMutation`, `useRemoveLastRoadMutation`** — TanStack Query mutation hooks, each calling the corresponding sidecar endpoint. On success they update `layoutResult[boundaryIndex]` via a shared helper; on error they leave the result unchanged (no optimistic state to unwind).
- **`DrawingToolbar.tsx`** — Drag ICR / Draw rectangle / Undo last buttons. All disabled when mode is `awaiting-ack` (with a "Syncing with engine…" subtitle). Undo-last shows stack depth in its label.
- **`App.tsx` wiring** — InteractionController instance via `useRef`, attached via `onMapReady`. Commit callbacks:
  1. Record `pendingCommit`, flip mode to `awaiting-ack` (preview stays visible, mode modules don't clear on commit).
  2. Fire the mutation with `onSettled: () => { clearDrawPreview(map); setMode('idle') }`. Success/error diverge only in the `layoutResult` swap (success) vs toast + stack no-op (error).
- **Style JSONs** — `kmz-draw-preview` + `kmz-draw-vertices` sources declared in both `pv-light.json` and `pv-dark.json` so theme swap repaints them for free via the existing `hydrateSources` pipeline. Three layers each: fill, dashed outline, vertex point.

**Unit tests (Phase 2):**
- `canvas/coords.test.ts` — 9 tests (Haversine accuracy at India lat, antimeridian, rect ring normalisation, ring centroid).
- `state/editingState.test.ts` — 8 tests (mode transitions, `awaiting-ack` preserves `pendingCommit`, undoStack LIFO, reset idempotence).

### LOC honesty

Phase 1: ~450 LOC (schemas + routes + 8 integration tests).
Phase 2: ~1,400 LOC across 13 new modules + 17 unit tests + ~60 LOC diff to App.tsx + style JSONs.
**S11 total: ~1,850 LOC** — in line with the 600-1000 estimate I gave in the S10.5 gate memo, adjusted for the stretch modes (polygon, line) not being implemented; if they were, total would land ~2,200.

## 3. Scope calls made during implementation

Documented here so they're not lost:

- **Single-boundary assumption.** `/add-road` commits broadcast to `boundaryIndex=0` only. Legacy PVlayout_Advance applied roads to ALL boundaries simultaneously. phaseboundary2.kmz is single-boundary so parity holds for the physical gate; multi-boundary broadcast is a small follow-up (repeat the mutation per boundary) that lands in S13.8's parity sweep or earlier if a multi-boundary fixture surfaces.
- **Stretch modes deferred.** Polygon + line modes are in the `EditingMode` union and have no-op branches in InteractionController; the mode-module implementations (polygonDraw, lineDraw) are not in Phase 2. DrawingToolbar shows rect + drag only. Reason: parity contract complexity (polygon close tolerance, vertex management, preview-segment rendering) adds ~200 LOC; rect + drag is the must-have per SPIKE_PLAN S11 amended scope.
- **Dim-on-drag polish deferred.** SPIKE_PLAN says "original ICR dim/dash during drag — optional polish". Not implemented in Phase 2; the user sees the original ICR rect alongside the translated preview (which is dashed-orange in light, dashed-amber in dark). Works, but not as crisp as legacy's semi-transparent override. Worth ~40 LOC in S11 polish follow-up.
- **Sidecar LA coverage non-determinism accepted.** `la_manager`'s step-2 coverage check iterates `placed_tables`; add-then-remove round-trips produce a slightly different LA set (and thus slightly different post-LA table count) than the initial /layout would. ~2% drift on phaseboundary2 (±6 tables of 611). Legacy has the same non-determinism. Documented in `test_s11_endpoints.py::test_remove_road_pops_last` with a tolerance check; S13.8 can tighten if needed.
- **No Ctrl+Z binding.** Matches legacy — "Undo last" is a button, not a keyboard shortcut. Could add later; deferred for now.

### Physical gate (Tauri dev app)

```bash
cd apps/desktop && bun run dev
```

In the Tauri window:

#### a. Boot + license + KMZ
- Enter your license key (Pro or Pro Plus recommended for the richest demo).
- ⌘O → `python/pvlayout_engine/tests/golden/kmz/phaseboundary2.kmz`.
- Click **Generate layout** with `Calculate cables` enabled.
- Layout renders: tables + 2 ICRs + inverters + cables + LAs.

#### b. Open DevTools console + enable probes
- `window.__S11_DEBUG__ = true` (or `VITE_INTERACTION_DEBUG=1` if you want it baked in at dev-server start).
- Filter for `[s11:` — probes should emit on every mode change, every commit, every sidecar call.

#### c. ICR drag happy path
- In the Layout tab, scroll to the new **Interaction** section. Click **Drag ICR**.
- Hover an ICR rectangle — cursor becomes `grab`.
- Click-and-drag an ICR to a new position, release.
- **Expected:**
  - Cursor tracks smoothly during drag (no jitter, no lag).
  - Orange dashed rectangle + centroid dot follow the cursor.
  - On release, the dashed preview **stays visible**. Mode buttons disable briefly with "Syncing with engine…" subtitle. Probe `[s11:sidecar] POST /refresh-inverters start`.
  - ~100-300ms later, preview clears, canvas swaps to new ICR position, tables under new ICR footprint cleared, inverter clusters + LAs updated. Probe `[s11:sidecar] POST /refresh-inverters end`.
- **Timing:** feels responsive. No "stuck syncing" state.

#### d. ICR drag abort (Escape)
- Start a drag, **press Escape before release**.
- Expected: preview clears immediately, mode returns to idle, original ICR unchanged, no sidecar call (probe log empty of new sidecar entries).

#### e. Rectangle draw happy path
- Click **Draw rectangle**.
- Cursor becomes crosshair.
- Click-drag a rectangle over some tables, release.
- **Expected:**
  - Rubber-band rect renders dashed-orange during drag.
  - On release, preview stays visible briefly, mode buttons disable with "Syncing…". Probe `[s11:sidecar] POST /add-road start`.
  - Response: tables inside the rect vanish, inverters re-cluster if applicable. `Undo last (1)` now shows `1` in DrawingToolbar.

#### f. Undo last obstruction
- Click **Undo last**.
- Expected: mode briefly `awaiting-ack`, sidecar returns. Tables under the removed rect come back (approximately — ±6 tables drift per §3 non-determinism note). Undo button becomes `Undo last` again (no count).

#### g. Draw 3 rects, undo 3 times
- Draw rect #1 over area A.
- Draw rect #2 over area B.
- Draw rect #3 over area C.
- **Expected:** Undo-last label shows `Undo last (3)`.
- Click Undo 3 times. Stack count goes 3 → 2 → 1 → 0. Areas restored in reverse order (LIFO).

#### h. Theme swap during drag
- Start a rect draw, mouse held down.
- Toggle theme in the status bar.
- **Expected:** style swap, preview re-paints in the new theme's colours (amber-dashed in dark), drag continues cleanly on release. Or: abort and reset mode to idle (acceptable per spec §10 error-handling rule). Probe `[s11:ctrl] styledata — re-attaching active mode`.

#### i. Sidecar error simulation (optional)
- Kill the sidecar process mid-session; try to drag an ICR.
- Expected: mutation fails, toast appears (or console error), preview clears, mode returns to idle.

#### j. New KMZ resets state
- ⌘O a different KMZ.
- Expected: undoStack cleared, mode = idle, all previous preview gone.

#### k. DevTools console clean
- Right-click → Inspect → Console.
- Only `[s11:*]` probes should appear during interactions.
- No errors or warnings during a typical drag + draw + undo cycle.
