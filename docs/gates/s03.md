## What was built

The sidecar now runs the real layout pipeline. `/parse-kmz`, `/layout`, and `/refresh-inverters` are implemented, wired to `pvlayout_core`, covered by tests, and verified to produce bit-for-bit (within tolerance) the same output as the PVlayout_Advance engine on three reference KMZs.

### 1. Adapters — `pvlayout_engine/adapters.py`

Translation layer between wire (pydantic) schemas and domain (dataclass) models. Every conversion is explicit — `params_to_core`, `result_from_core`, plus `*_to_core` variants for `/refresh-inverters`'s round-trip. Enum values match by string; `LayoutResult.usable_polygon` is dropped on the way out (shapely not JSON-serializable).

### 2. Geometry helper — `pvlayout_engine/geometry.py`

`reconstruct_usable_polygon(result, perimeter_road_width)` rebuilds the shapely `usable_polygon` from the result's persistent fields (`boundary_wgs84`, `obstacle_polygons_wgs84`, `placed_roads`, `utm_epsg`). Used by `/refresh-inverters` which receives a stateless result and needs the geometry back.

Noted limitation (documented inline): line obstructions from the original KMZ are NOT yet round-trippable on the wire — they're consumed during `/layout` and their setback is baked into `usable_polygon` at that time. If S11 introduces obstruction editing that touches line obstructions, we'll extend the schema then.

### 3. Routes — `pvlayout_engine/routes/layout.py`

Three real endpoints (replacing the S2 echo routes):

| Endpoint | Body | Response | Role |
|---|---|---|---|
| `POST /parse-kmz` | multipart file | `ParsedKMZ` | Uploads a KMZ, returns boundaries + obstacles + line obstructions + centroid. |
| `POST /layout` | `LayoutRequest { parsed_kmz, params }` | `LayoutResponse { results }` | Full pipeline: `run_layout_multi` → `place_lightning_arresters` → `place_string_inverters`. Order matches PyQt app's `_refresh_inverters`. |
| `POST /refresh-inverters` | `RefreshInvertersRequest { result, params }` | `LayoutResult` | Rebuilds `usable_polygon`, reruns LA + string-inverter placement only. |

All gated by the bearer-token dependency from S2. S2's `/_schemas/echo/*` routes are gone.

### 4. Golden-file harness

**Three reference KMZs** delivered by you, under `python/pvlayout_engine/tests/golden/kmz/`:
- `Kudlugi Boundary (89 acres).kmz` — 6-boundary multi-phase plant
- `phaseboundary.kmz` — single boundary, moderate complexity
- `phaseboundary2.kmz` — single boundary with obstacles + line obstruction

**Capture script** at `scripts/capture_golden.py`: runs the domain logic directly (no HTTP, no PyQt) with default `LayoutParameters`, serialises each `LayoutResult` list to `tests/golden/expected/<stem>.json`. Re-run this script whenever the baseline should legitimately move.

**Parity test** at `tests/golden/test_layout_parity.py`: one test per KMZ. Uploads the file via `/parse-kmz`, calls `/layout`, compares to baseline with:
- **Exact match** on counts (tables, ICRs, string inverters, LAs), boundary names, UTM EPSG, integer indices.
- **Position tolerance** ≤ 0.01 m on all x/y coordinates (tables, ICRs, inverters, LAs, tables_pre_icr, cable endpoints).
- **Relative float tolerance** ≤ 1e-6 on capacities, areas, ratios, cable lengths.

### 5. Bug fix surfaced by this spike (S1 regression)

**What happened.** My S1 sed pattern for rewriting imports from `{core,models,utils}.X` → `pvlayout_core.{core,models,utils}.X` anchored at start-of-line (`^`). This caught all 16 top-level imports I'd identified — but missed **8 indented imports** inside function bodies. The missed sites imported lazily (often inside conditionals or functions) and so passed the S1 smoke test (which only exercised module-level imports). The bug detonated on `phaseboundary2.kmz` because it has a line obstruction (TL corridor), which exercises an indented `from utils.geo_utils import utm_to_wgs84` at `layout_engine.py:97`.

**Fix applied in S3.** Broadened the sed pattern to match leading whitespace:
```
^([[:space:]]*)from {core,models,utils}\.  →  \1from pvlayout_core.{core,models,utils}.
```
Applied across all of `pvlayout_core/`. All 8 indented sites rewritten. Post-fix grep confirms zero remaining stale imports.

**Preventive additions to S3:**
- The golden harness now exercises code paths including line obstructions (phaseboundary2.kmz).
- The integration test `test_layout_rejects_empty_boundaries` ensures we still get meaningful errors when input is degenerate.
- `capture_golden.py` run from scratch (with PyInstaller-level deps) re-validated that no other stale imports remain.

**Why it didn't merit its own ADR.** This was a mechanical correctness bug in the S1 import rewrite — not a design decision. Fixed in place with a backward-compatible extension to the rewrite logic.

## Known limitations (by design)

- **Inverter placement silent on defaults.** With `enable_cable_calc = False` (the baseline), `place_string_inverters` does not populate `placed_string_inverters`. This matches PyQt5 behavior and is revisited in S10.
- **Line obstructions are wire-invisible.** After `/layout` the line obstructions' effect is baked into `usable_polygon` (which we don't serialize). A client that calls `/refresh-inverters` will see a polygon rebuilt from obstacles + roads only. For the PyQt-parity workflow this is acceptable; if S11 changes it we'll extend the schema.
- **`/refresh-inverters` not yet exercised by golden tests.** It's covered by integration tests for auth + schema validity. End-to-end behavior testing lands when the React frontend starts using it (S11).
- **Multi-boundary KMZ + obstacle cases that crash inside `run_layout`** now return an `ERROR:` boundary_name instead of raising. That already matched PyQt behavior; just calling it out.

## Decisions made during this spike

1. **Stateless sidecar.** `/refresh-inverters` takes the full prior result and rebuilds `usable_polygon` on each call rather than keeping a session. Trades a small compute cost for zero server-side state. Correct given our single-user / single-desktop deployment model.
2. **Golden capture uses the domain logic directly, not the HTTP surface.** Keeps the baseline independent of any HTTP / adapter bug — if a wire-format regression ever hid a real layout regression, the golden test would still catch the real issue because the expected values came straight from `run_layout_multi`.
3. **Position tolerance of 0.01 m, relative float tolerance 1e-6.** Matches the spike plan's stated tolerance. Wide enough to absorb platform-dependent floating-point rounding, tight enough that any real geometric drift is caught.
4. **`capture_golden.py` does NOT auto-run in CI.** Baselines are human-approved artefacts; regenerating them on every test defeats their purpose.
