## What was built

### 1. Sidecar — `placed_tables_wgs84` + `placed_icrs_wgs84`

[`schemas.py`](../../python/pvlayout_engine/pvlayout_engine/schemas.py): `LayoutResult` gains two new fields holding pre-projected WGS84 corner rings (closed 5-tuple per object) for placed tables and ICRs.

[`adapters.py`](../../python/pvlayout_engine/pvlayout_engine/adapters.py): `result_from_core` populates them via a new local `_rect_corners_wgs84` helper that uses `pvlayout_core.utils.geo_utils.utm_to_wgs84`. Same corner-order convention as `kmz_exporter._table_corners_wgs84` (BL → BR → TR → TL → BL).

The desktop renders polygons directly from these fields — no client-side projection needed. Aligns with [ADR-0002](../adr/0002-no-basemap.md) (canvas owns the projection responsibility).

New test: [`tests/integration/test_layout_wgs84_corners.py`](../../python/pvlayout_engine/tests/integration/test_layout_wgs84_corners.py) — verifies field presence, length parity with `placed_tables` / `placed_icrs`, ring closure, WGS84 range, and centroid sanity.

### 2. `@solarlayout/sidecar-client` — types + `runLayout()`

[`packages/sidecar-client/src/index.ts`](../../packages/sidecar-client/src/index.ts):
- `LayoutResult` interface gains `placed_tables_wgs84` and `placed_icrs_wgs84`.
- New `LayoutRequest` / `LayoutResponse` envelopes.
- New `runLayout(parsedKmz, params): Promise<LayoutResult[]>` method on `SidecarClient`. JSON POST to `/layout`; unwraps the `{ results }` envelope server-side.

(Layout-related types — `LayoutParameters`, `ModuleSpec`, etc. — landed in S8.8.)

### 3. Frontend deps

`apps/desktop/package.json`:
- `react-hook-form@^7.73.1`

### 4. Zod schema + state extension

[`apps/desktop/src/state/layoutParams.ts`](../../apps/desktop/src/state/layoutParams.ts) gains `layoutParametersSchema` — Zod object mirroring `LayoutParameters` exactly, with min/max ranges from the PyQt5 reference inventory. A compile-time `_ZodMatchesType` assertion forces drift between the Zod schema and the TS interface to surface as a TypeScript error.

[`apps/desktop/src/state/useLayoutMutation.ts`](../../apps/desktop/src/state/useLayoutMutation.ts) — TanStack Query `useMutation` wrapping `sidecar.runLayout`. On success, hydrates `useLayoutResultStore`. Per [ADR-0003](../adr/0003-state-architecture.md): server cache lives in TanStack Query; the snapshot of "current layout result" lives in Zustand; this hook bridges them.

### 5. UI primitive — `LockedSectionCard`

[`packages/ui/src/compositions/LockedSectionCard.tsx`](../../packages/ui/src/compositions/LockedSectionCard.tsx) — affordance for tier-gated features. Used by S9 in the Energy tab; reusable for any future PRO_PLUS lock state.

### 6. Compositions — `LayoutPanel` + `SummaryPanel`

[`apps/desktop/src/panels/LayoutPanel.tsx`](../../apps/desktop/src/panels/LayoutPanel.tsx) — react-hook-form bound to `useLayoutParamsStore`, validated by `layoutParametersSchema`. Five sections matching the PyQt5 reference (layout-relevant subset only):
- **Module:** length, width, wattage.
- **Table:** orientation (Portrait / Landscape `Segmented`), modules per row, rows per table, table gap.
- **Spacing & tilt:** override-toggle + value for tilt (`Switch` + conditional `NumberInput`); same for row pitch. The `null` sentinel = auto-derive on the sidecar.
- **Site:** perimeter road width.
- **Inverter sizing:** design mode (`Select`), max strings per inverter / SMB (label switches with mode), max SMB per central (only visible in central mode), enable-cable-calc toggle.

Generate button at the bottom — `disabled` when no project loaded or mid-mutation; copy adapts (`"Open a KMZ to generate"` / `"Generate layout"` / `"Generating…"`).

[`apps/desktop/src/panels/SummaryPanel.tsx`](../../apps/desktop/src/panels/SummaryPanel.tsx) — three states:
- Generating: skeleton bars in the StatGrid + PropertyRow list.
- No result yet: small placeholder copy.
- Has result: `StatGrid` with MWp / Tables / ICRs + `PropertyRow`s for plant area, used area, packing density, row pitch, tilt angle. Aggregates across boundaries.

### 7. Two-tab inspector IA

[`apps/desktop/src/App.tsx`](../../apps/desktop/src/App.tsx) — `InspectorSkeleton` removed. Inspector now wraps a Radix `Tabs` with two tabs:
- **Layout** — fully built. `LayoutPanel` + `SummaryPanel`.
- **Energy yield** — `LockedSectionCard` placeholder ("Available in PRO_PLUS"). Body lands in S13. The IA is locked now so S13 doesn't redesign the panel mid-spike.

### 8. MapCanvas extension — tables + ICRs + ICR labels

[`packages/ui/src/compositions/MapCanvas.tsx`](../../packages/ui/src/compositions/MapCanvas.tsx):
- Two new props: `tablesGeoJson?` and `icrsGeoJson?`.
- New `icrLabels?: IcrLabel[]` prop — renders text overlays as absolutely-positioned divs anchored via `map.project()`. Updated on `move` / `zoom` events. Sidesteps the glyph/sprite stack we don't have. Pragmatic for ~1–5 ICRs per typical plant; deck.gl `TextLayer` is the answer when N grows (S10 territory).
- `hydrateSources` refactored into a single `SourceData` parameter — cleaner as we add more layers.

[`apps/desktop/public/map-styles/pv-light.json`](../../apps/desktop/public/map-styles/pv-light.json) and [`pv-dark.json`](../../apps/desktop/public/map-styles/pv-dark.json):
- Two new sources: `kmz-tables`, `kmz-icrs`.
- Four new layers: `tables-fill`, `tables-outline`, `icrs-fill`, `icrs-outline`. Tables get a subtle bluish-gray fill + thin outline (engineering-tool restraint). ICRs get a stronger amber fill + clearer outline (they're discrete buildings, want them to stand out from the table grid).

### 9. App.tsx integration

- New `layoutGeoJson` memo — recomputes from the layout-result store; passed to MapCanvas.
- `clearLayoutResult()` called on every new KMZ open so stale tables don't overlay a new plant.
- `useLayoutMutation` wired; Generate button calls `handleGenerate` which mutates with current `layoutParams`.
- Layout-error path — renders the existing `OpenErrorOverlay` with retry → `handleGenerate`.
