## What was built

### 1. `@solarlayout/entitlements-client` (TypeScript)

Previously an S0 stub. Now a real package:

- **Zod schemas** (`src/types.ts`) mirror the live mvp_api `/entitlements` and `/usage/report` contracts — `{ success, data: { user, plans[], licensed, availableFeatures, totalCalculations, usedCalculations, remainingCalculations } }`.
- **HTTP client** (`src/client.ts`) exposes `createEntitlementsClient({ baseUrl, fetchImpl, timeoutMs })` with `getEntitlements(key)` and `reportUsage(key, feature)`. Errors collapse to `EntitlementsError(status, message, body)` — `status=0` for network failures, numeric statuses for HTTP errors. Validates responses against the Zod schemas at the boundary.
- **License-key helpers** — `LICENSE_KEY_PATTERN`, `isPlausibleLicenseKey(s)` for the `sl_live_` prefix check.
- **Tests**: 14 Bun tests in `src/client.test.ts` covering successful parse, Bearer auth header, custom base URL, 401/5xx/timeout/network/schema-invalid paths, and `/usage/report` request/response shape.

### 2. Rust keyring bridge

- **`apps/desktop/src-tauri/src/keyring.rs`** — wraps the `keyring` v3 crate with `apple-native` / `linux-native` / `windows-native` features. Service `solarlayout-desktop`, account `license_key`.
- **Tauri commands**: `get_license()`, `save_license(key)`, `clear_license()` — registered in [`lib.rs`](../../apps/desktop/src-tauri/src/lib.rs) via `invoke_handler`.
- Errors stringified and surfaced verbatim to the frontend; `NoEntry` on `get_license` returns `Ok(None)`, `clear_license` is idempotent.

### 3. Sidecar session enforcement

- **`pvlayout_engine/session.py`** — `SessionState` dataclass (thread-safe via `threading.Lock`) holds `available_features: set[str]` + `plan_name: str | None` + `initialized: bool`. `require_feature(key)` is a FastAPI dependency factory returning 503 before the shell has pushed entitlements, 403 when the feature key is missing, and passes through otherwise.
- **`pvlayout_engine/routes/session.py`** — `POST /session/entitlements` (body: `{ available_features: [], plan_name: str|null }`) replaces the sidecar's session state; `GET /session` reads it back.
- **Wiring** — `server.build_app` creates a `SessionState` and attaches it to `app.state.session`; `/session` + `/session/entitlements` routes sit behind the existing bearer-token dependency alongside `/health`, `/parse-kmz`, `/layout`, `/refresh-inverters`.
- **Tests**: 9 new integration tests in `tests/integration/test_session.py`:
  - Initial state is `initialized: false`.
  - POST updates, GET reflects.
  - Second POST replaces rather than unions.
  - 401 without bearer token.
  - Unknown fields rejected (422).
  - `require_feature` returns 503 before init, 403 on missing feature, 200 on present feature, 503 after `clear`.

### 4. React auth layer

- **`apps/desktop/src/auth/licenseKey.ts`** — thin wrapper around Tauri keyring commands. Preview-mode bypass: outside Tauri, `getLicenseKey()` returns the sentinel `sl_live_preview_mode_design_review_only` so design reviewers see the populated shell rather than the blocking dialog.
- **`apps/desktop/src/auth/useEntitlements.ts`** — TanStack Query hook + sidecar push:
  - `useEntitlementsQuery(key)` — single-shot fetch, session-long stale time, no retry (online-required per ADR 0001), uses `tauriFetch` inside Tauri and `globalThis.fetch` in preview.
  - `useSyncEntitlementsToSidecar(entitlements, sidecar)` — fire-and-forget `POST /session/entitlements` when entitlements change; skipped in preview (`port === 0`).
  - Preview-mode bypass: the hook returns `PREVIEW_ENTITLEMENTS` (all features, Free plan, 95/100 remaining) for the sentinel key.
- **`apps/desktop/src/auth/EntitlementsProvider.tsx` + `FeatureGate.tsx`** — context + wrapper component. `<FeatureGate feature="dxf">…</FeatureGate>` renders children when `availableFeatures` includes the key, fallback otherwise. `useHasFeature(key)` exposes the same check as a hook. Consumed in S10+ when feature-gated UI actually ships.

### 5. Dialogs

- **`src/dialogs/LicenseKeyDialog.tsx`** — modal with two modes:
  - `first-launch` — dismiss-proof (no Esc, no outside-click).
  - `change` — dismissible via Cancel.
  - Password-masked input, `sl_live_` prefix validation, "Don't have a key? Sign up for free →" link opens `https://solarlayout.in/sign-up` via the Tauri shell plugin, and a "Verifying…" state while the parent's entitlements query runs.
- **`src/dialogs/LicenseInfoDialog.tsx`** — read-only modal showing account (name, email), quota aggregate + progress bar, per-plan cards with feature chips + per-plan quotas, and Change Key / Clear License / Close actions.

### 6. TopBar + shell integration

- **[`packages/ui/src/compositions/TopBar.tsx`](../../packages/ui/src/compositions/TopBar.tsx)** — added `onViewLicense` prop; the avatar DropdownMenu's "View license" item now fires it.
- **[`apps/desktop/src/App.tsx`](../../apps/desktop/src/App.tsx)** — rewritten as a clear state machine:

  | Phase | Render |
  |---|---|
  | `sidecar-booting` | Splash |
  | `sidecar-error` | Splash + `BlockingOverlay` |
  | `loading keyring` | Splash "Loading…" |
  | `no-license` | Splash + blocking `LicenseKeyDialog(mode=first-launch)` |
  | `validating` | Splash "Verifying licence…" (+ the dialog in submitting mode if user just entered a key) |
  | `license-error` (network / 5xx) | Splash + `BlockingOverlay` with Retry |
  | `license-error` (401) | keyring cleared, drops back to `no-license` with inline error |
  | `ready` | Full shell + `Chip` reads `plans[0].planName` |

- **Chip** (previously hardcoded "Pro") now renders the plan name from entitlements verbatim — typically "Free" for new signups.
- **Avatar initials** now derive from `entitlements.user.name`; e.g. "Arun Patra" → "AP", falls back to "--" if absent.
- **Account menu handlers**: View license → opens `LicenseInfoDialog`; Clear license → keyring delete + state reset + shell back to blocking dialog.
- **Command palette**: new entries for `View licence` and `Clear licence`.

### 7. `main.tsx` — QueryClientProvider

TanStack Query client wraps the app under ThemeProvider/TooltipProvider. Defaults set to `staleTime: Infinity, gcTime: Infinity, retry: false, refetchOnWindowFocus: false, refetchOnReconnect: false` per ADR 0001 (no background refetch; session-long cache only).

`core:default` (already present from S6's drag fix) is sufficient. The three new commands `get_license` / `save_license` / `clear_license` are auto-allowed as app-local commands (no extra ACL needed; custom commands are allowed by default when registered in `invoke_handler`).

## Deviations / explicit decisions recorded here

1. **No offline grace window.** Replaces the aspirational cached-entitlements + N-day-grace design from the earlier SPIKE_PLAN draft. See ADR 0001. Surfaces: Human Gate step 4 is now "wifi off → error banner with Retry", not "wifi off → app still works".
2. **Edition terminology retired at the app boundary.** `pvlayout_core/edition.py` stays verbatim (vendored, never modified), but the shell consumes feature-key strings from `availableFeatures` directly — no `Basic/Pro/Pro Plus` enum in TS/React code. The chip literal is `plans[0].planName` verbatim (today likely "Free"), to be reviewed in S13.7 when subscription tiers are designed.
3. **Sidecar gets entitlements from the shell (push), not from `api.solarlayout.in` directly.** Simpler, single network-dependency path, supports mid-session updates. Defense-in-depth trust model: the sidecar's enforcement is UX hygiene; revenue protection is in the server-side `/usage/report` quota decrement. See `pvlayout_engine/session.py` doc comment for the full rationale.

## Usability follow-ups noted during the gate (to revisit in a later spike)

- **Validation-error surface offers only "Retry", no "Enter different key".** When the stored key returns 401 (revoked / rotated), the current flow drops back to the first-launch dialog automatically — good. But when the entitlements call fails with a non-401 error (network, 5xx, scope-blocked URL), the BlockingOverlay shows only a "Retry" button. There's no affordance to pop the LicenseKeyDialog and try a different key, even if the user suspects their stored key is part of the problem.
- **Offline error message is technical.** With wifi off, the BlockingOverlay reads `error sending request for url (https://api.solarlayout.in/entitlements)` verbatim from the Tauri http plugin. Should be a plain-English message like "Couldn't reach the SolarLayout servers — check your internet connection and try again." Keep the raw detail accessible via DevTools for support cases.
- **macOS Keychain prompts on first read/write.** Expected macOS behaviour — when Rust's `keyring` crate touches a new Keychain entry, macOS asks for the user's login password to authorise access. Happens first time in dev mode AND first time in release mode (different signing identities → different perceived-caller identity). Becomes a one-time prompt per install once we sign with a stable Developer ID in S14. Worth a tooltip in the license dialog explaining the prompt.

Non-blocking for S7 (primary paths all pass). Revisit in S13.7 (subscription redesign will touch these surfaces) or a dedicated UX cleanup spike.
