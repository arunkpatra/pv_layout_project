## What was built

### 1. Sidecar — WGS84 projections for 5 new geometry kinds

[`schemas.py`](../../python/pvlayout_engine/pvlayout_engine/schemas.py): `LayoutResult` gains five new fields holding pre-projected WGS84 geometries — same pattern S9 introduced for tables/ICRs:

- `placed_string_inverters_wgs84: list[list[UTMPoint]]` — closed 5-point rect rings
- `dc_cable_runs_wgs84: list[list[UTMPoint]]` — polylines (uses `route_utm` when set, falls back to `[start, end]`)
- `ac_cable_runs_wgs84: list[list[UTMPoint]]` — polylines
- `placed_las_wgs84: list[list[UTMPoint]]` — closed 5-point rect rings
- `placed_las_circles_wgs84: list[list[UTMPoint]]` — 65-point circles (64 segments + closing) sampled at `la.radius` metres from the LA centre

[`adapters.py`](../../python/pvlayout_engine/pvlayout_engine/adapters.py): adds two helpers — `_polyline_wgs84` and `_circle_ring_wgs84` — and populates the five new fields in `result_from_core`. Rect rings reuse the existing `_rect_corners_wgs84` helper from S9.

Tests:
- [`tests/smoke/test_adapters_s10.py`](../../python/pvlayout_engine/tests/smoke/test_adapters_s10.py) — 6 unit tests that exercise `result_from_core` with a synthetic `core.LayoutResult`. Covers: rect ring closure, cable polyline projection, `route_utm` fallback to `[start, end]`, LA rect ring, LA circle ring structure, circle radius accuracy (haversine, <1% tolerance), and empty-core-emits-empty-lists safety.
- [`tests/integration/test_layout_s10_wgs84.py`](../../python/pvlayout_engine/tests/integration/test_layout_s10_wgs84.py) — one integration test that runs `/parse-kmz` → `/layout` (cable_calc OFF for speed) and confirms the new fields exist in the response with correct length parity on LA rects + circles.

**Why a synthetic unit test instead of an end-to-end `enable_cable_calc=True` integration test?** Cable routing on phaseboundary2 is O(minutes) — too slow for CI. The unit test covers the adapter precisely (100% of the new field surface), and the integration test covers the real-response shape. The split keeps the S10 test time under 2 seconds total.

### 2. `@solarlayout/sidecar-client` — 5 new `LayoutResult` fields

[`packages/sidecar-client/src/index.ts`](../../packages/sidecar-client/src/index.ts): mirrors the pydantic additions. TypeScript-level drift between the two is a compile error.

### 3. Frontend — `layoutToGeoJson` extended to 8 FeatureCollections

[`apps/desktop/src/project/layoutToGeoJson.ts`](../../apps/desktop/src/project/layoutToGeoJson.ts) adds:

- `stringInverters: FeatureCollection<Polygon>`
- `dcCables: FeatureCollection<LineString>`
- `acCables: FeatureCollection<LineString>`
- `las: FeatureCollection<Polygon>` (rects)
- `laCircles: FeatureCollection<Polygon>` (protection zones)

Cable `LineString` coordinates come straight from `dc_cable_runs_wgs84` / `ac_cable_runs_wgs84` — no client-side projection. Degenerate runs (`< 2` points) are skipped defensively.

Tests ([`layoutToGeoJson.test.ts`](../../apps/desktop/src/project/layoutToGeoJson.test.ts)): +2 new cases — populated S10 layers and degenerate cable-line filtering.

### 4. MapLibre styles — 5 new sources + 8 new layers per theme

[`apps/desktop/public/map-styles/pv-light.json`](../../apps/desktop/public/map-styles/pv-light.json) and [`pv-dark.json`](../../apps/desktop/public/map-styles/pv-dark.json):

- 5 new sources: `kmz-string-inverters`, `kmz-dc-cables`, `kmz-ac-cables`, `kmz-las`, `kmz-la-circles`.
- 8 new layers in paint order (bottom → top): `dc-cables` · `ac-cables` · `string-inverters-fill` · `string-inverters-outline` · `la-circles-fill` · `la-circles-outline` · `la-rects-fill` · `la-rects-outline`.
- Initial `visibility: "none"` on `ac-cables` + all four LA layers — matches PVlayout_Advance defaults. MapCanvas overrides based on the visibility store at paint time.

Colours chosen for engineering-tool restraint: inverters purple (distinct from ICR amber + table blue-grey), cables desaturated but clearly visible over tables, LA rects saturated red (safety-equipment convention), LA circles heavily translucent so tables remain legible through them.

### 5. Layer-visibility state — new Zustand slice

[`apps/desktop/src/state/layerVisibility.ts`](../../apps/desktop/src/state/layerVisibility.ts) — `useLayerVisibilityStore` slice per [ADR-0003](../adr/0003-state-architecture.md). Two flags (`showAcCables`, `showLas`), two setters, one `resetToDefaults` action. Defaults: both off (PVlayout_Advance match).

Why cross-component client state (not `useState`)? The toggle writer (`VisibilitySection`) and the reader (`MapCanvas` via `App.tsx`) are sibling components. Per ADR-0003, that's the Zustand-slice case. Additive for future visibility toggles (S11 drawing overlays, S12 export preview).

Tests ([`layerVisibility.test.ts`](../../apps/desktop/src/state/layerVisibility.test.ts)): 4 tests covering defaults, independent setter behaviour, reset.

### 6. MapCanvas — 5 new geojson props + 2 visibility props

[`packages/ui/src/compositions/MapCanvas.tsx`](../../packages/ui/src/compositions/MapCanvas.tsx):

- 5 new optional `FeatureCollection` props (`stringInvertersGeoJson`, `dcCablesGeoJson`, `acCablesGeoJson`, `lasGeoJson`, `laCirclesGeoJson`).
- 2 new boolean props (`showAcCables`, `showLas`) that drive `setLayoutProperty(layerId, "visibility", ...)` on:
  - `showAcCables` → `ac-cables`
  - `showLas` → `la-rects-fill` + `la-rects-outline` + `la-circles-fill` + `la-circles-outline`
- `applyVisibility()` helper called from (a) the map's `load` handler (initial paint), (b) the `styledata` handler after theme swap (new style JSON resets layer visibility), and (c) the visibility effect on toggle change. Idempotent.
- Existing `propsRef` + `lastAppliedStyleUrl` patterns preserved — no regression risk on the S8 cascade-layer fix or the S9 theme-toggle fix.

### 7. `VisibilitySection` — canvas layer toggles inside the Layout tab

[`apps/desktop/src/panels/VisibilitySection.tsx`](../../apps/desktop/src/panels/VisibilitySection.tsx) — new inspector section rendered between `LayoutPanel` and `SummaryPanel` when a layout exists (`{layoutResult && <VisibilitySection />}` in `App.tsx`). Two `ToggleRow`s:

- **Show AC cables** — gated on `cables` feature key
- **Show lightning arresters** — gated on `cables` feature key (shared with AC cables — both are "PRO-tier" features; splits cleanly in S13.7 if a dedicated `lightning_arresters` key is introduced)

When the user's `availableFeatures` doesn't include `cables`, the Switch renders `disabled` and a `<Chip tone="accent">Pro</Chip>` appears inline with the label. Matches the "disabled + upgrade badge for Basic tier" spec.

Tests ([`VisibilitySection.test.tsx`](../../apps/desktop/src/panels/VisibilitySection.test.tsx)): 6 RTL tests covering both the entitled + basic-tier paths — click behaviour, store dispatch, disabled state, chip presence/absence. Closes the "VisibilitySection behaviour" half of the S9 RTL-gap backlog item.

### 8. SummaryPanel — 7 new rows

[`apps/desktop/src/panels/SummaryPanel.tsx`](../../apps/desktop/src/panels/SummaryPanel.tsx):

- Second `StatGrid` row: **Modules** · **Inverters** · **LAs** — visible whenever a layout exists (the S9 backlog module-count item lives here).
- New PropertyRows (conditional on non-zero value):
  - **Inverter capacity** (kWp)
  - **DC cable length** — auto-units `m` / `km` by magnitude
  - **AC cable length** — same
- PRO_PLUS-gated rows (conditional on `energy` feature-key membership):
  - **AC capacity** (MW)
  - **DC/AC ratio**

Aggregation: counts and lengths sum across boundaries; `dcAcRatio` reports the first boundary's value (it's a plant-level invariant, not additive — same convention as `rowPitchM` and `tiltAngleDeg` in S9).

### 9. App.tsx wiring

- `useLayerVisibilityStore` subscribed (`showAcCables`, `showLas`, `resetToDefaults`).
- `handleOpenKmz` resets visibility to defaults on every new KMZ load — mirrors S9's pattern for `layoutParams` + `layoutFormKey` (fresh start = fresh visibility state).
- MapCanvas receives the 5 new geojson props + 2 visibility booleans.
- Inspector tree: `<LayoutPanel />` → `<VisibilitySection />` (if `layoutResult`) → `<SummaryPanel />`.
