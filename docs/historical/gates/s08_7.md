## Goal

Stand up a Vitest + React Testing Library harness for the frontend (desktop + UI package), wire CI to run lint / typecheck / test / build + sidecar pytest on every push, and prove the harness with a small but real test suite covering the existing testable surfaces.

## In scope

### Frontend test harness

- **Vitest** as the test runner. Aligns with Vite (already our build tool); zero config friction.
- **`@testing-library/react` + `@testing-library/jest-dom` + `@testing-library/user-event`** ‚Äî standard RTL stack.
- **`happy-dom`** as the DOM environment (faster than jsdom for non-canvas tests; we'll add `jsdom` later if a specific test needs full DOM fidelity).
- Per-package `test` scripts that turbo runs in parallel.
- Test files co-located with source: `Foo.tsx` ‚Üî `Foo.test.tsx`. Matches `CLAUDE.md` ¬ß11 convention.
- A `apps/desktop/src/test-setup.ts` that imports `@testing-library/jest-dom`, mocks `window.matchMedia` (for `useTheme`), and stubs `__TAURI_INTERNALS__` so `inTauri()` returns false in tests by default.

### Mocked sidecar client

- A factory `createMockSidecarClient(overrides)` in `apps/desktop/src/test-utils/mockSidecar.ts` that returns a `SidecarClient` shape with vi.fn() implementations. Tests can override per-method behavior. Typed against the real `SidecarClient` interface so drift surfaces as a TS error.

### Initial test coverage (proof-of-life suite)

Not exhaustive ‚Äî just enough to prove the harness works on representative surfaces. Roughly 8‚Äì12 tests:

- **`packages/ui`:**
  - `Button` ‚Äî renders all variants, dispatches `onClick`, respects `disabled`.
  - `NumberInput` ‚Äî emits onChange with parsed number, clamps to min/max.
  - `Switch` ‚Äî controlled toggle behavior.
  - `kmzToGeoJson` (the pure function we hand-tested in S8) ‚Äî closed-ring boundary, unclosed-ring boundary, empty input, ring-with-2-points (degenerate), obstacle filtering. **High value: this is the function that almost shipped a bug in S8.**
  - `MapCanvas` ‚Äî verifies the cascade-layer fix doesn't regress: render with a stub `maplibregl` mock, assert containerRef receives className `"w-full h-full"` not `"absolute inset-0"`.
- **`apps/desktop`:**
  - `App` mounted with a healthy mock sidecar + valid entitlements: shell renders, theme toggle works, palette opens on `‚åòK`.
  - `App` with a 401 from entitlements: license dialog appears.

These exercise: rendering, interaction, mocked async (TanStack Query), pure logic, regression guards.

### CI

- A single `.github/workflows/ci.yml` that triggers on:
  - `push` to any branch except `main` (avoids double-runs after PR merge).
  - `pull_request` targeting `main`.
- Jobs:
  - **`frontend`** ‚Äî checkout ‚Üí setup-bun ‚Üí `bun install --frozen-lockfile` ‚Üí `bun run lint && bun run typecheck && bun run test && bun run build`.
  - **`sidecar`** ‚Äî checkout ‚Üí setup-uv ‚Üí `cd python/pvlayout_engine && uv sync --extra dev && uv run ruff check && uv run mypy pvlayout_engine && uv run pytest tests/ -q`.
- Both jobs run in parallel. Status checks bubble to PR view.
- No artifact upload, no caching beyond what bun/uv handle natively. Keep CI minimal and fast.

### Lint / typecheck unification

- Replace the `[S5 stub] echo` lines in `apps/desktop/package.json` and `packages/ui/package.json` `lint` scripts with real ESLint runs (using the shared config in `packages/eslint-config` if it exists, otherwise inline a sensible TS+React rule set).
- Confirm `typecheck` is real (it already is ‚Äî runs `tsc --noEmit` everywhere).

## Out of scope

- **Visual regression tests** ‚Äî Percy / Chromatic / Playwright snapshot. Adds CI cost and image-diff noise; defer until S13.5 (dark theme parity needs visual comparison anyway).
- **End-to-end Tauri tests** ‚Äî would require building the Tauri binary in CI and driving the webview. Major lift for marginal coverage on top of unit tests. Defer to a later release-engineering spike.
- **Sidecar coverage uplift** ‚Äî pytest already exists from S3. Not adding new tests in this spike beyond what's already there.
- **Story / docgen tooling** (Storybook / Ladle) ‚Äî design-system tooling, not test harness. Considered for S13.5 polish, not now.
- **Pre-commit hooks** (husky / lint-staged) ‚Äî local DX, not CI; nice-to-have but not required by S8.7.

## Notes / decisions

- **happy-dom over jsdom:** ~3√ó faster on small suites; sufficient fidelity for everything we test except canvas. If a future test needs canvas rendering, switch that file to jsdom or just stub maplibre-gl in the mock.
- **No coverage thresholds** in CI yet. Premature optimization; we'd be coverage-gating ourselves before the suite is big enough to reflect quality. Revisit when suite > 100 tests.
- **No snapshot tests** in the initial suite. Snapshots become brittle when design churns; we'll selectively introduce them only for stable serializations (e.g., `kmzToGeoJson` output structure for a fixed input).
- **CI on push-to-non-main** rather than push-to-main: PRs are the merge gate; main commits are squash-merges of already-CI'd PR work. Saves duplicate runs.
